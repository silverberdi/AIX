# AIX Product Constitution

1. AIX is governance-first.
2. AIX is document-centric.
3. AIX is metadata-first.
4. AIX is opinionated but extensible.
5. AIX is document-event-driven.
6. AIX is policy-driven.
7. AIX separates authentication from authorization.
8. AIX separates Platform runtime from Business runtime.
9. AIX is AI-ready, not AI-dependent.
10. AIX uses declarative, typed policies instead of executable scripts.
11. AIX uses contract-first APIs.
12. AIX uses role/workspace-driven UX.
13. AIX is an Event-Driven Modular Monolith.
14. AIX keeps SharedKernel minimal.
15. AIX uses one database per tenant.
16. AIX treats documents as governed evidence, not files.
17. AIX separates document identity from operational context.
18. AIX uses PrimeNG/Avalon before custom UI.
